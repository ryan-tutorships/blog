## Como funciona

<CardGrid stagger>
  <Card title="1) Faça o trabalho" icon="open-book">
    Resuma um artigo, evolua um projeto pessoal, contribua em OSS, ou publique um post técnico com conteúdo.
  </Card>

<Card title="2) Envie a comprovação" icon="seti:image">
  Submeta um link + screenshot. Uma evidência por submissão, com dedupe por URL quando aplicável.
</Card>

  <Card title="3) Ganhe pontos e badges" icon="approve-check">
    Pontos têm limite semanal para evitar farming. Badges destacam consistência, shipping e impacto.
  </Card>
</CardGrid>

## O que vale pontos

<CardGrid stagger>
  <Card title="Resumos" icon="open-book">
    TL;DR + 3 takeaways + o que você faria diferente. Ideal: 1 página.
  </Card>

<Card title="Projetos" icon="jetbrains">
  Feature enviada com prova (deploy, release tag, demo). Pequeno e finalizado &gt; grande e
  abandonado.
</Card>

<Card title="LinkedIn" icon="linkedin">
  Post com substância: problema → abordagem → resultado. Sem “post vazio” valendo ponto.
</Card>

<Card title="Comunidade / Mentoria" icon="comment">
  Ajudou alguém? Registre o contexto e a solução. Ajuda rastreável &gt; “te ajudei”.
</Card>

<Card title="Open Source" icon="github">
  PR mergeado, issue bem reportada ou triagem útil. Prova: link + status.
</Card>
</CardGrid>

## Regras rápidas

- **Revisão:** em até **72h**.
- **Sem edição silenciosa:** totais vêm de um **ledger append-only**.
